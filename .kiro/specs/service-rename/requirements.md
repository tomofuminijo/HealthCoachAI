# 要件書

## はじめに

HealthCoachAI サービスを Healthmate-CoachAI に名前変更し、Healthmate プロダクトの命名規則との一貫性を確保する。この変更により、プロダクト全体でのブランディングと識別性を向上させる。

## 用語集

- **HealthCoachAI**: 現在のAI健康コーチサービス名
- **Healthmate-CoachAI**: 新しいサービス名
- **サービス名**: コード、設定ファイル、ドキュメント内で使用される識別子
- **プロダクト名**: Healthmate プロダクト全体の名称
- **AWS リソース**: CloudFormation、IAM、AgentCore等のAWSサービスリソース
- **設定ファイル**: .bedrock_agentcore.yaml、requirements.txt等の設定ファイル
- **ドキュメント**: README.md、コメント、ステアリングファイル等

## 要件

### 要件 1

**ユーザーストーリー:** 開発者として、サービス名を Healthmate-CoachAI に変更したいので、プロダクト全体での命名一貫性を保てる

#### 受け入れ基準

1. WHEN ファイル名やディレクトリ名を変更する THEN システムは新しい名前 Healthmate-CoachAI を使用する
2. WHEN 設定ファイルを更新する THEN システムは新しいサービス名を反映する
3. WHEN AWS リソース名を変更する THEN システムは healthmate-coach-ai プレフィックスを使用する
4. WHEN ドキュメントを更新する THEN システムは新しいサービス名を一貫して使用する
5. WHEN コード内の参照を更新する THEN システムは新しい名前空間を使用する

### 要件 2

**ユーザーストーリー:** 運用担当者として、既存のデプロイメントが影響を受けないようにしたいので、段階的な移行ができる

#### 受け入れ基準

1. WHEN 名前変更を実行する THEN システムは既存のデプロイメントを破壊しない
2. WHEN 新しい設定でデプロイする THEN システムは古いリソースをクリーンアップする手順を提供する
3. WHEN 移行プロセスを実行する THEN システムは各ステップの検証方法を提供する
4. WHEN ロールバックが必要な場合 THEN システムは元の状態に戻す手順を提供する

### 要件 3

**ユーザーストーリー:** 他のサービス開発者として、統合ポイントが正しく更新されることを確認したいので、サービス間連携が継続する

#### 受け入れ基準

1. WHEN Healthmate-HealthManager サービスから参照する THEN システムは新しいサービス名を使用する
2. WHEN HealthmateUI サービスから参照する THEN システムは新しいサービス名を使用する
3. WHEN ステアリングファイルを更新する THEN システムは新しいサービス名を反映する
4. WHEN 環境変数を更新する THEN システムは新しい命名規則を使用する

### 要件 4

**ユーザーストーリー:** 開発者として、開発環境とテスト環境で新しい名前が正しく動作することを確認したいので、品質を保証できる

#### 受け入れ基準

1. WHEN テストスクリプトを実行する THEN システムは新しいサービス名で正常に動作する
2. WHEN デプロイメントスクリプトを実行する THEN システムは新しい名前でリソースを作成する
3. WHEN 手動テストを実行する THEN システムは新しいサービス名で応答する
4. WHEN ログを確認する THEN システムは新しいサービス名を出力する
